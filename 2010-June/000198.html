Dear All,
I have been attempting to produce the Sphere/Diplane/Helix decomposition 
using a Radarsat-2 Quad-pol image. Unfortunately RAT has crashed at the 
end of the process. I have tried the process on two Window's machines 
(oh, the shame) running XP Prof. and Vista. I am using RAT 0.21.

The XP version either produces no change (first attempt) or crashes 
after showing an error message stating the program reached the end of 
the (temp) file workfile1.rat (following attempts)
The Vista procedure gives a RAT error message saying that the end of 
file was encountered in workfile1.rat

In both cases all I have done is open the Radarsat-2 image as an 
external file, saved it as a .rat file, calibrated it using the Quegan 
approach, transformed it to circular basis and then attempted the SDH 
decomposition. I should point out that the transformation to circular 
basis looks suspiciously like the lexicographic basis.

Is this a widespread bug?
