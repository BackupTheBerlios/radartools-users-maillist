Hi Ian,

yes, indeed, there was a bug if used with full-polar 4-channel data. It is corrected now for the future release[*].
But for now, if you want to use it, you can apply "PolSAR->Calibration->Cross-polar symmetrization" after the cross-talk calibration, but before polarization basis change to circular. Then it will work fine.
Thanks for the bug info.
