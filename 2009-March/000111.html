Hi,

The complex values stored in the COSAR file are actually the  
DN(Digital Numbers) recorded. There is a nice discussion about  
converting and calibrating data on the Infoterra site.
http://www.infoterra.de/fileadmin/Verzeichnisordner/Dokumente/2_AboutUs/0207_FormalDocs/TSXX-ITD-TN-0049-radiometric_calculations_I1.00.pdf

The constants are stored in the metadata XML which RAT can partially  
understand. The mechanisms for using the orbits etc are not there yet  
but I can try to implement. Incidence angle calculations will also  
require an underlying DEM.

I hope this helps let me know if there are any specific feature requests.
