Hi Waleed,

according to the code, you get back the "unwrapped phase", i.e. a real value
which is not bound to [-pi, pi]. It is a relative phase offset with respect
to the lowest value in the image which is set to 0.
If you want to know the topographic variations in meters, you need to scale
with the wavenumber.
