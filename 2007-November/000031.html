Hi Garrett,

It looks like a variable wrapping problem at the  2GB limit. [*]
Please copy the attached modified file "cut_region.pro" into the
directory "rat/general/" of your rat installation (this change will be
included in the upcoming new release).
This should solve this problem.
Since we didn't work a lot with files >2GB it is possible, that there
are still some other small bugs with reference to large files.
Thanks for reporting any bug.

Best regards,
Maxim

[*] assuming your system and your IDL version allow to access files
greater than 2GB, which is now standard. Either this is possible
depends on the system-constant !VERSION.FILE_OFFSET_BITS. If it's
equal 64 then it's ok, if it is only 32, then you are in trouble.
